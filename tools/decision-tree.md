# Decision Tree — แยก Type 1 vs Type 2

ใช้ classify ประเภทการตัดสินใจ

---

## Step 1: Reversibility (พลิกกลับได้ไหม?)

```
การตัดสินใจนี้ ถ้าเลือกผิด พลิกกลับได้ง่ายไหม?
│
├── พลิกกลับได้ยาก หรือไม่ได้เลย → Type 1 (คิดหนัก)
│   ตัวอย่าง: ซื้อบ้าน, แต่งงาน, มีลูก, ลาออกจากงานประจำ
│   → ใช้ Pre-Mortem, วิเคราะห์ละเอียด
│
├── พลิกกลับได้ แต่มีต้นทุน → Type 1.5 (คิดหนักปานกลาง)
│   ตัวอย่าง: เปลี่ยนสายอาชีพ, ย้ายเมือง, เรียน ป.โท
│   → วิเคราะห์ 5 มิติ + Pre-Mortem แบบย่อ
│
└── พลิกกลับได้ง่าย → Type 2 (ลงมือทันที)
    ตัวอย่าง: ซื้อของ, ลองร้านอาหารใหม่, เรียนคอร์สสั้น
    → ใช้ Fail-Fast Protocol
```

## Step 2: Cost of Failure (ต้นทุนถ้าพลาด)

```
ถ้าเลือกผิด — ต้นทุนเท่าไร?
│
├── สูงมาก (เสียเงินมาก, เสียเวลา > 1 ปี, เสียโอกาสสำคัญ)
│   → Type 1 แม้จะพลิกกลับได้
│
├── ปานกลาง (เสียเงินบ้าง, เสียเวลา 1-6 เดือน)
│   → Type 1.5
│
└── ต่ำ (เสียเงินนิดหน่อย, เสียเวลา < 1 เดือน)
    → Type 2
```

---

## Step 3: Energy Budget (พลังในการคิด)

```
คุณมีพลังงานสมองพอที่จะคิดเรื่องนี้ลึกไหม?
│
├── มี Energy สูง → คิดเต็มที่ ใช้ทุกเครื่องมือ
├── มี Energy ปานกลาง → ใช้แค่ 5-D + Action Step
└── Energy ต่ำ
    ├── เรื่องสำคัญ → พักก่อนแล้วค่อยคิด
    └── เรื่องไม่สำคัญ → สุ่มหรือเลือกอันแรก
```

---

## Step 4: Decision Matrix

| Reversibility | ต้นทุนต่ำ | ต้นทุนปานกลาง | ต้นทุนสูง |
|--------------|----------|--------------|----------|
| **พลิกง่าย** | Type 2 — ลงมือทันที, สุ่มได้ | Type 2 — คิดนิดหน่อย | Type 1.5 — คิดหน่อย |
| **พลิกยาก** | Type 1.5 — คิดหน่อย | Type 1 — คิดหนัก | Type 1 — คิดหนักมาก |
